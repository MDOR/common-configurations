# Prettier
Prettier helps to keep code standardized and consistency across teams and within the project, according to a list of rules defined for the team

## prettierrc
If not present, create a `.prettierrc` and include the following information.
```json
{
  "arrowParens": "avoid",
  "bracketSpacing": true,
  "htmlWhitespaceSensitivity": "css",
  "insertPragma": false,
  "jsxBracketSameLine": false,
  "jsxSingleQuote": true,
  "printWidth": 80,
  "proseWrap": "preserve",
  "quoteProps": "as-needed",
  "requirePragma": false,
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "none",
  "useTabs": false,
  "overrides": [
    {
      "files": [".prettierrc", ".huskyrc", ".lintstagedrc", ".babelrc", ".eslintrc"],
      "options": {
        "parser": "json"
      }
    },
    {
      "files": ["*.js", "*.jsx", "*.mjs", ".vue"],
      "options": {
        "parser": "babel",
        "trailingComma": "es5",
        "vueIndentScriptAndStyle": false
      }
    },
    {
      "files": ["*.ts", "*.tsx"],
      "options": {
        "parser": "typescript",
        "trailingComma": "es5"
      }
    }
  ]
}
```
